Diagnostic solaire : arbre de decision

Objectif

Quand on charge les donnees d’une installation, l’app doit determiner comment obtenir une serie temporelle PV fiable. Plusieurs sources existent, plusieurs tests permettent de valider leur coherence, et les resultats de ces tests orientent vers la bonne strategie.

Ce document presente l’arbre de decision complet sous forme de diagramme Mermaid, suivi des explications detaillees de chaque test et de chaque deduction.

Arbre de decision

Les tests, un par un

Test 1 : Coherence de perimetre (nocturne)

Quoi : la nuit (22h-5h), il n’y a pas de PV. Le bilan energetique doit se verifier exactement :

elec_consumption = offtake - feedin   (la nuit)

Comment : on calcule ecart = elec - offtake + feedin sur tous les pas nocturnes. La mediane de cet ecart doit etre proche de zero.

Deductions :

Resultat Interpretation Action
Mediane < 0.05 kWh Les deux sources mesurent le meme perimetre Le bilan PV est fiable
Mediane > 0.05 kWh Perimetres differents ou desynchronisation Ne pas utiliser le PV par bilan, passer a Elia
Quelques ecarts > 1 kWh Spikes ORES ponctuels (pas un probleme de perimetre) Activer le filtre outliers

Test 2 : Stabilite PV (ratio mensuel)

Quoi : si le perimetre est OK, on reconstitue le PV par bilan et on compare chaque mois au profil Elia (1 kWc). Le ratio donne le kWc equivalent.

Comment : coefficient de variation (CV) des ratios mensuels.

Deductions :

Resultat Interpretation Action
CV < 30% Installation PV a capacite constante kWc fixe, PV reconstitue fiable sur toute la periode
CV > 30% Capacite PV a change dans le temps Utiliser kWc adaptatif (mediane glissante)
kWc croissant mois apres mois Mise en service progressive Restreindre ou scaler par mois
kWc decroissant Degradation, ombrage saisonnier, ou panne partielle Investiguer

Test 3 : Autoconsommation PV

Quoi : quelle part du PV produit est consommee sur place (vs injectee) ?

Comment : taux = (PV_reel - feedin) / PV_reel

Deductions :

Resultat Interpretation Consequence pour la simulation
~0% PV et PAC sur circuits separes, ou PAC s’arrete quand PV produit L’optimiseur a peu de marge pour augmenter l’autoconsommation
20-60% PV alimente partiellement l’installation L’optimiseur peut decaler la PAC vers les heures PV
> 80% Installation deja bien optimisee (surplus PV rare) Gains marginaux attendus
Negatif Artefact de spikes ORES (feedin surestime) Activer le filtre outliers

Test 4 : Puissance PAC (via COP)

Quoi : en separant les periodes COP=0 (PAC off) et COP>0 (PAC on), on estime la puissance de la PAC et le talon de l’installation.

Comment : P95 = quantile(elec - talon quand COP>0, 0.95) * 4

Deductions :

Resultat Interpretation Action
P95 coherent avec la plaque (±20%) Donnees fiables, bonne separation PAC/talon OK
P95 >> puissance nominale Elec_consumption inclut d’autres grosses charges Investiguer, le talon est sous-estime
P95 << puissance nominale La PAC tourne rarement a pleine puissance Normal en mi-saison
Talon > 5 kW Installation avec charges permanentes elevees (eclairage, chaufferie) Normal pour du collectif

Le kWc adaptatif : de l’estimation a la serie PV

Une fois le kWc estime (par la mediane glissante), on l’utilise pour produire la serie temporelle PV du site :

PV_site(t) = PV_elia_1kwc(t) * kWc(t)

Ou kWc(t) est la valeur de la mediane glissante au jour de t.

Cela donne une serie : - Lisse (pas de spikes ORES) - Adaptee a la taille reelle de l’installation a chaque instant - Meteorologiquement correcte (Elia reflete les conditions reelles) - Utilisable pour la simulation sur toute la periode
